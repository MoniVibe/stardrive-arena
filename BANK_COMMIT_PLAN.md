# Passive Visual Bank Commit Plan

Branch: `fix/bank`

## Scope

Implemented a passive-authoritative, visual-only banking layer for ship mesh and tactical icon presentation. The change does not write `Ship.YRotation`, `Ship.XRotation`, `[StarData]`, SX payload fields, durable digest fields, or authoritative hash fields from the passive visual path.

## Files changed

- `Ship_Game/Ships/Ship.cs`
  - Added private non-`[StarData]` passive visual bank state.
  - Added SX tick/rotation observation, signed delta calculation, clamped target calculation, render-refresh easing, stale/stop decay, and internal test accessors.
- `Ship_Game/Multiplayer/Authoritative/Authoritative4XSession.cs`
  - `ApplyShipTransformLine()` now observes SX tick/rotation deltas for visual bank targets before applying the host-authored transform.
- `Ship_Game/Ships/Ship_Update.cs`
  - Passive scene-object sync now uses `YRotation + passiveVisualBank` for the render matrix only.
- `Ship_Game/Ships/Ship_Rendering.cs`
  - Tactical icon draw path consumes the same render-only bank as a small icon lean.
- `UnitTests/Multiplayer/Authoritative4XSessionTests.cs`
  - Added `Authoritative4XPassiveVisualBank_IsRenderOnlyAndDigestNeutral_Headless`.
- `DESYNC_ORIENTATION_FINDINGS.md`
  - Updated with implementation and verification notes.

## Proof target

`Authoritative4XPassiveVisualBank_IsRenderOnlyAndDigestNeutral_Headless` forces a host-authored SX `Rotation` delta while host `YRotation`/`XRotation` remain zero. It applies the snapshot through the passive replica, refreshes the passive scene object, verifies the render-only mesh/icon visual bank is nonzero, then captures the passive client snapshot and asserts both `SyncDigest` and `TransformDigest` match the host. It also verifies visual-bank decay after rotation deltas stop.
